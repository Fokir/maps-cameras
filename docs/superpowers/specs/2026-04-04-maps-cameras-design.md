# Maps Cameras — Спецификация

Веб-сервис для отображения камер видеонаблюдения на карте с просмотром RTSP-стримов через go2rtc.

## Архитектура

```
┌─────────────────────────────────────────────┐
│              Браузер (React SPA)             │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │  Map     │ │ Stream   │ │   Editor     │ │
│  │ (Leaflet)│ │ (WebRTC/ │ │ (формы, DnD, │ │
│  │          │ │  MSE)    │ │  undo/redo)  │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
│                    │                         │
│              Zustand Store                   │
│         (cameras, editor, ui)                │
└────────────────┬───────────────────┬─────────┘
                 │ REST API          │ WS (сигналинг +
                 │                   │ heartbeat)
                 ▼                   ▼
        ┌────────────────┐   ┌──────────────┐
        │   Go Backend   │   │   go2rtc      │
        │   :8080        │   │ (Frigate)     │
        │                │   │ server.local  │
        │  • CRUD камер  │   │  • WebRTC     │
        │  • SQLite      │   │  • MSE/WS     │
        │  • Статика SPA │   │  • RTSP proxy │
        │  • Прокси go2rtc│   │              │
        └────────────────┘   └──────────────┘
```

### Ключевые решения

- **Go-бэкенд** — единый бинарник, раздаёт React SPA через `embed.FS`, API для камер, проксирует сигналинг к go2rtc.
- **go2rtc** — уже развёрнут через Frigate на `http://server.local`. Наш сервис не запускает свой экземпляр.
- **Стримы по требованию** — RTSP-поток добавляется в go2rtc только при просмотре, удаляется при закрытии.
- **WebRTC + MSE fallback** — основной протокол WebRTC (~0.5с задержка), fallback на MSE (~1-2с) при неудаче.
- **Без аутентификации** — сервис в локальной сети.

## Режимы работы

### 1. Режим просмотра (по умолчанию)

Layout: `[Карта | ↔ | Стрим]` — resizable split, 50/50 по умолчанию.

- Карта отображает все камеры в виде иконок с полупрозрачным конусом обзора.
- Клик по камере → камера становится активной → справа открывается видеопоток.
- Переключатель слоя карты (улицы / спутник) — внизу справа на карте.
- Кнопка перехода в режим редактирования — внизу слева на карте.

### 2. Режим редактирования

Layout: `[Список камер ~200px | Карта (flex) | Форма ~280px]`

#### Список камер (левая панель)
- Перечень всех камер с цветовым индикатором.
- Камеры, уже размещённые на карте, отмечены визуально.
- Drag-and-drop камеры из списка на карту для привязки к позиции.
- Кнопка "Добавить" внизу — открывает пустую форму справа.

#### Карта (центр)
- Все камеры отображаются с конусами обзора.
- Клик по камере — появляются контролы:
  - **Поворот** — drag камеры для вращения направления конуса.
  - **Угол обзора** — drag верхней/нижней ручки конуса.
  - **Дистанция** — drag средней ручки (дальняя сторона конуса).
- Кнопка "✎ Ред." рядом с камерой — открывает предзаполненную форму справа.
- Любое изменение сохраняется сразу (debounced PUT запрос).

#### Форма (правая панель)
- Поля: название, RTSP URL, цвет (палитра).
- Кнопки: сохранить, удалить.

#### Undo/Redo
- При входе в режим редактирования — сохраняется начальный снапшот всех камер.
- Каждое изменение (перемещение, поворот, угол, дистанция, создание, удаление, правка свойств) → новый снапшот в стек.
- Ctrl+Z — откат, Ctrl+Shift+Z — повтор.
- При выходе из режима редактирования — стек очищается.

### Мобильное представление

- Карта на весь экран.
- Клик по камере → полноэкранный стрим с кнопкой "назад к карте".

## Камера на карте

Каждая камера отображается как:
- Иконка (круг) с заданным цветом камеры.
- Полупрозрачный конус обзора — треугольник от точки камеры.

Параметры конуса:
- **rotation** (0-360°) — направление камеры.
- **angle** (10-180°) — ширина конуса.
- **distance** — длина конуса в метрах.

## Карта

- **Библиотека:** Leaflet + react-leaflet.
- **Тайлы улиц:** OpenStreetMap.
- **Тайлы спутника:** Esri World Imagery (`https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}`).
- **Масштаб:** зум 18-20 (небольшая территория — здание/двор).
- **Начальный центр:** определяется по расположению камер (fit bounds), или из конфига при первом запуске.
- **Центрирование при resize:** карта сохраняет текущий центр при изменении размера окна.

## Модель данных

```
Camera {
  id:         string    (UUID)
  name:       string    // "Вход №1"
  rtsp_url:   string    // "rtsp://192.168.1.10/stream1"
  color:      string    // "#f7768e"
  lat:        float64   // позиция на карте (null если не размещена)
  lng:        float64
  rotation:   float64   // направление камеры (0-360°)
  angle:      float64   // угол обзора конуса (10-180°)
  distance:   float64   // дистанция конуса в метрах
  created_at: timestamp
  updated_at: timestamp
}
```

## API

### Камеры

| Метод  | Путь               | Описание          |
|--------|--------------------|--------------------|
| GET    | `/api/cameras`     | Список всех камер  |
| POST   | `/api/cameras`     | Создать камеру     |
| PUT    | `/api/cameras/:id` | Обновить камеру    |
| DELETE | `/api/cameras/:id` | Удалить камеру     |

### Стриминг

| Метод   | Путь                  | Описание                                  |
|---------|-----------------------|-------------------------------------------|
| POST    | `/api/stream/start`   | Добавить стрим в go2rtc, вернуть URL      |
| POST    | `/api/stream/stop`    | Убрать стрим из go2rtc                    |
| GET/WS  | `/api/stream/webrtc`  | Проксирование WebRTC сигналинга к go2rtc  |
| GET/WS  | `/api/stream/ws`      | Проксирование MSE WebSocket к go2rtc      |

### Импорт камер

| Метод | Путь                  | Описание                                |
|-------|-----------------------|-----------------------------------------|
| POST  | `/api/cameras/import` | Импорт камер из M3U плейлиста (файл)   |

**Формат M3U:**
```m3u
#EXTM3U
#EXTINF:-1,Вход №1
rtsp://192.168.1.10/stream1
#EXTINF:-1,Парковка
rtsp://192.168.1.11/stream1
```

- Парсинг `#EXTINF` — извлекается имя камеры из поля после запятой.
- Следующая строка — RTSP URL.
- Камеры создаются без позиции на карте (lat/lng = null), с дефолтным цветом.
- После импорта пользователь размещает камеры на карте через drag-and-drop в режиме редактирования.
- Дубликаты по RTSP URL пропускаются.
- Ответ: `{imported: N, skipped: N, errors: [...]}`.

В UI — кнопка "Импорт M3U" в режиме редактирования (рядом с "Добавить" или в меню).

### Жизненный цикл стрима

1. Фронтенд вызывает `POST /api/stream/start {camera_id}`.
2. Бэкенд берёт RTSP URL из БД, добавляет стрим в go2rtc (`PUT /api/streams`).
3. Бэкенд возвращает `{stream_name, webrtc_url, ws_url}`.
4. Фронтенд подключается через WebRTC (или MSE fallback).
5. Поддерживается WebSocket-соединение: при обрыве — бэкенд удаляет стрим из go2rtc.
6. Heartbeat как fallback: ping каждые 10с, cleanup через 30с без ответа.

## Тулинг и сборка

### Фронтенд
- Vite + React + TypeScript
- Zustand (состояние)
- react-leaflet (карта)
- Tailwind CSS (стили)

### Разработка
- `Makefile`: `dev-frontend`, `dev-backend`, `build`, `run`
- Vite dev server (`:5173`) проксирует `/api/*` на Go (`:8080`)
- Go hot reload через `air`

### Production
```
make build
  → npm run build (Vite → web/dist/)
  → go build -o maps-cameras cmd/server/main.go
  → один бинарник со встроенной статикой
```

### Docker
- Multi-stage: Node (сборка фронтенда) → Go (сборка бэкенда) → минимальный образ.
- `docker-compose.yml` только для нашего сервиса.
